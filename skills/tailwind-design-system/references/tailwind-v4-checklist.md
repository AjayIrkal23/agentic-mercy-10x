# Tailwind v4 Checklist

Use this reference only when the task already routed into `tailwind-design-system`.

## Confirm Before Editing
- Verify the project is actually on Tailwind CSS v4.
- Inspect current theme and token files before adding new ones.
- Check whether a shared component primitive already exists.

## Implementation Areas
- Theme tokens and CSS variable strategy.
- Shared utility conventions.
- Reusable component primitives and variants.
- Dark-mode and responsive behavior.
- Migration notes when replacing older Tailwind patterns.

## Validate Before Finishing
- Shared tokens remain coherent across screens.
- Components still match the chosen visual direction.
- Accessibility and responsive behavior are preserved.
- Exact version-sensitive syntax has been checked against current docs.
