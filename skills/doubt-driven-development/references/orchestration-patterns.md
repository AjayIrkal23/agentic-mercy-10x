# Adversarial-review orchestration patterns

- Fresh-context review: hand the decision to a clean reviewer with no priors.
- Steelman then attack: state the strongest case, then try hardest to break it.
- Cost-of-verification test: verify now if cheaper than debugging later.
- Escalate scrutiny with stakes (production, security, irreversible ops).
- Record the doubt + resolution so the decision is auditable.
